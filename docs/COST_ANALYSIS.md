# A 股盘面监控 MVP 成本与可行性调研

调研日期：2026-07-21
适用范围：两名用户、朋友自用、A 股信息与风险提醒，不自动交易、不公开分发行情。
重要提示：所有数字都是估算值，不等于实际账单。没有购买、充值、开通套餐或修改外部账号。

## 结论先行

推荐先验证“全市场实时日线快照，每 5 分钟拉一次，在本系统保存快照并计算 5 分钟变化”。Tushare 官方价目显示，其个人版全市场实时日线为 200 元/月，每分钟最多 50 次且单次可提取全市场；实时分钟行情则为 1000 元/月。对于只做风险提醒的两人自用工具，前者很可能已足够，预计每月可少 800 元行情费。

当前不建议直接购买。产品已接入零费用的真实实验源，用来呈现六个主要指数、关注股票、历史日线、市场宽度、行业/概念和全市场当日限价字段。2026-07-21 已验证限价覆盖 5199/5200 只沪深股票，但这些公开页面接口没有正式 API 授权或生产 SLA，只适合私有 MVP；正式全市场能力仍应使用供应商试用或最小权限验证完整性、延迟、授权和限频。

2026-07-21 再次核对 Tushare 官方接口：[`rt_min`](https://tushare.pro/document/2?doc_id=374) 提供实时 1/5/15/30/60 分钟数据但需要单独权限；[`rt_sw_k`](https://tushare.pro/document/2?doc_id=417) 的申万行业实时行情也是独立权限；[`stk_limit`](https://tushare.pro/document/2?doc_id=183) 和 [`trade_cal`](https://tushare.pro/document/2?doc_id=26) 均标注 2000 积分门槛。当前没有用户批准的 Token 或付费权限，因此这些能力保持“尚未开通”。

### 实验候选，不作为默认生产源

- 腾讯、新浪和东方财富公开行情页面接口：2026-07-21 已读取六个指数、沪深市场宽度、5199/5200 只股票限价字段、申万一级行业、概念排行/目录与指数历史成交额；无需账号、暂未产生费用。它们没有面向本产品的正式 API 授权、生产 SLA 或稳定字段文档，因此仅在私有 MVP 中标注“真实数据·实验源”，不得公开分发行情，也不能替代正式授权数据源。
- AKShare：大量接口来自公开网页，免费但上游字段、反爬和授权随时变化，存在封禁、稳定性与再分发风险。
- Baostock：适合免费的历史/日线实验，不满足稳定的盘中实时监控。
- 直接抓取财经网站页面或未公开接口：不采用。没有明确授权、容易变更或封禁，也不适合云端周期调用。

## 2. 请求规模与降本假设

A 股连续交易时间按 240 分钟/日估算，按 22 个交易日/月：

| 扫描周期 | 盘中扫描/日 | 盘中扫描/月 | 全市场原始快照存储/月 | 400 只板块成分/月 | 30 只自选/月 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 5 分钟 | 48 | 1,056 | 约 929 MB | 约 67.6 MB | 约 5.1 MB |
| 30 分钟 | 8 | 176 | 约 155 MB | 约 11.3 MB | 约 0.8 MB |
| 60 分钟 | 4 | 88 | 约 77 MB | 约 5.6 MB | 约 0.4 MB |
| 120 分钟 | 2 | 44 | 约 38.7 MB | 约 2.8 MB | 约 0.2 MB |
| 180 分钟 | 2 | 44 | 约 38.7 MB | 约 2.8 MB | 约 0.2 MB |

假设每行标准化快照约 160 字节、全市场 5500 只。真实值取决于字段和压缩；生产应只长期保留聚合指标，原始高频快照设置 7–30 天滚动保留。

### 三种请求范围

1. 全市场快照：一次拿齐市场宽度、涨跌停与个股，最省 API 次数，但网络和存储最大；Tushare 实时日线允许每次提取全市场，是当前最有价值的验证路径。
2. 关注板块成分股：约 200–800 只，费用和存储显著下降，但无法可靠判断“全市场下跌家数/跌停数”。
3. 仅自选股：成本最低，但失去市场和板块背景，只适合作为降级模式。

推荐：盘中保留“全市场聚合指标 + 关注对象明细”，不长期保存所有股票的每个 5 分钟原始行。

## 5. 大模型

- [OpenAI 官方说明](https://help.openai.com/en/articles/6950777-what-is-chatgpt-plus)：ChatGPT Plus 为 20 美元/月的 ChatGPT 订阅，API 使用单独计费，不能把 Plus 会话当后台 API。
- [GPT-5 mini 官方页](https://developers.openai.com/api/docs/models/gpt-5-mini)：输入 0.25 美元/百万 tokens，缓存输入 0.025 美元/百万，输出 2 美元/百万。
- 估算每天一次收盘复盘，输入 6000、输出 1500 tokens，22 个交易日约 0.099 美元/月；按估算汇率 7.20 折合约 0.72 元/月。
- 本机发现 OpenAI-compatible 中转配置，但中转价格、授权和 SLA 未知，且最小连通测试也可能产生费用，因此未调用、未复制密钥。

盘中每 5 分钟都调用模型会让成本和故障面同时放大，没有必要。数值与风险级别全部由代码计算；模型只在收盘后写解释，模型失败则输出数字型复盘。

## 7. 购买前的验证清单

1. 供应商书面确认：两人自用、云函数定时拉取、保存衍生指标、私有页面展示是否在个人授权范围。
2. 用试用账号验证一个交易日：09:30、午休、15:00 数据时间；全市场一次返回数量；停牌/ST/科创/创业板/北交所涨跌停价。
3. 测量 5 分钟全市场快照的响应时间、失败率、实际字节数与日调用量。
4. Server酱先用试用/免费额度完成测试消息，确认个人微信能收到，再决定是否年付。
5. CloudBase 先看账号是否有免费体验环境和 3000 点/月，确认超额计费开关，禁止默认开启自动续费或按量付费。
6. 模型中转先确认单价、数据处理和日志保留，再用最小请求测试；设置每日 0.20 美元硬上限。
