# Aria 监盘

> A 股盯盘 for Aria

一个给两名普通用户使用的手机优先 A 股盘面风险提醒与每日复盘 MVP。所有股票数据、预警和结论只能来自真实行情；数据缺失时明确显示不可用。不自动交易、不构成投资建议，也不会在未经确认时产生行情、云资源、通知或模型费用。

## 主源码仓库

- GitHub [willwang2528/a-shares](https://github.com/willwang2528/a-shares) 是本项目后续开发的唯一主源码仓库。
- `main` 是稳定分支；开发前先同步 `origin/main`，完成测试后再推送 GitHub。
- GPT Sites 只是私有部署目标，不是主源码仓库。每次发布必须使用已推送到 GitHub 的确切提交。
- 云数据库内容、Sites 访问权限和服务端 Secret 不存入 GitHub。

## 现在能做什么

- 首页查看开市/休市、六个主要指数、上一交易日的真实全市场收盘复盘，以及只从本人关注范围生成的“和我有关的三件事”。
- 关注页可以从真实在线目录搜索、选择并保存行业、概念或 A 股股票；股票读取真实快照，失败时明确显示不可用。
- “有色金属”等行业大类只作为一组公司的分类展示，不显示虚构的单一价格。
- 复盘页可以选择日期，读取当前关注股票的上一交易日收盘、当日开盘、收盘、最高、最低和成交量，并保存查询缓存。
- 上一交易日复盘覆盖沪深成交额及前日对比、涨跌家数、真实涨跌停价统计、行业/概念方向和主要承压方向；字段缺失时显示“没有数据”。
- 独立保存周期监控、关键时点和每日复盘计划；周期支持 5/30/60/120/180 分钟。当前部署尚未连接后台定时触发器，关闭网页后不会自动执行。
- 选择“只推异常 / 每次都摘要 / 摘要 + 异常”三种模式。
- 保存设置、预警、任务记录、通知日志与收盘复盘。
- 在成本中心查看当日官方价格、调用量、存储量和三个成本方案。
- 配置 Server酱或邮件后，从服务端发送真实通知；未配置时明确显示待配置。

## 电脑小白本地启动

1. 安装 Node.js 22 或更高版本。
2. 打开本项目文件夹。
3. 第一次运行：`npm install`
4. 启动：`npm run dev`
5. 浏览器打开终端显示的地址，通常是 `http://localhost:3000`
6. 结束时在终端按 `Control + C`。

所有真实密钥都只能放在 `.env`，可以复制 `.env.example` 后填写。不要修改 `.env.example`，不要把 `.env` 发给别人或提交到 Git。

## 第一次使用

1. 打开“任务”，分别确认三条任务链开关。
2. “多久扫描一次”建议先保留 5 分钟；“什么情况下推送”建议先选“只推异常”。
3. 打开“关注”，搜索行业分类或股票，从多个候选中选中一个，再确认保存。
4. 打开“预警”，点“运行一次扫描”；只有真实数据达到阈值才会生成记录。
5. 打开“复盘”，选择日期并点“读取这一天”。交易日显示真实股票数据，休市日明确显示“没有数据”。
6. 打开“成本”，确认尚未批准的付费能力。

## 测试通知

### 零账号测试：本机浏览器通知

1. 打开“通知”。
2. 选“本机浏览器通知”。
3. 点“测试本机通知”。
4. 浏览器询问权限时选“允许”。
5. 看到系统通知表示这台设备的前台通知可用。它不是可靠的后台 Web Push。

### 个人微信：Server酱

1. 打开 [Server酱官网](https://sct.ftqq.com/)。
2. 用希望接收消息的微信扫码登录。
3. 按官网步骤设置消息通道并复制 SendKey。
4. 在本地 `.env` 或云平台服务端 Secret 中设置 `SERVERCHAN_SENDKEY`。不要粘贴到页面、聊天或代码。
5. 重启或重新部署。
6. 在“通知”选“Server酱”，保存，点“测试当前服务端渠道”。
7. 确认微信收到“Aria 监盘测试通知”，再决定是否购买订阅。

免费版每天最多 5 条，不适合开启“每次都摘要”的 5 分钟周期。

## 当前真实数据范围

- 数据源：腾讯公开行情页面接口、新浪财经板块目录/排行、东方财富板块目录与指数历史日线，均标注为“真实数据·实验源”。
- 当前覆盖：上证指数、深证成指、创业板指、科创 50、沪深 300、中证 1000；关注股票当前快照；最多 20 只关注股票的前复权历史日线。
- 首页全市场复盘覆盖沪市/深市成交额、前一交易日同口径成交额、上涨/下跌/平盘家数，以及约 5200 只沪深股票的真实当日涨停价和跌停价字段。
- 行业使用新浪申万一级成分股汇总；概念复盘使用新浪真实概念排行；关注搜索额外读取东方财富公开概念目录，当前可检索约 495 个真实概念代码。
- 交易时段读取最新盘中快照；休市时显示最近交易日收盘快照和明确时间。
- 历史复盘的“开盘前基准”是上一交易日收盘价，“开盘后”是 09:30 开盘价与当日收盘价。当前不提供 09:25 集合竞价或完整历史分钟线。
- 同一用户、同一日期、同一组关注股票的历史复盘会写入 D1 缓存：过去交易日缓存 7 天，当日数据缓存 5 分钟，“没有数据”缓存 1 小时。
- 页面启动时先显示上次成功读取并带时间戳的真实快照，再在后台刷新。
- 该接口无需账号、当前不会产生费用，但没有面向本产品的正式授权文件或生产 SLA，只能作为私有 MVP 的实验数据源。
- 如果读取失败，页面会显示失败原因或“没有数据”，不生成新的预警或结论。

## 正式全市场行情接入前

先阅读 [成本与可行性调研](docs/COST_ANALYSIS.md)。当前推荐候选是 Tushare 全市场实时日线快照 200 元/月，但必须先确认云端定时调用、两人自用展示与衍生数据保存是否符合授权。不要先付款再确认授权。

## 调度心跳

后台只需要一个高频心跳，请求 `POST /api/scheduler` 并携带服务端 `Authorization: Bearer <SCHEDULER_SECRET>`。它会根据数据库设置、交易日、交易时段和上次运行时间判断哪些任务到期。午休、周末和节假日会写明跳过原因。

当前私有部署尚未连接 CloudBase 定时触发器。需要用户确认 CloudBase 环境后，再把这个入口连接到托管定时触发器。

## 常见问题

**页面为什么显示“没有数据”？**
当前真实数据源没有提供该字段、网络读取失败或数据日期不一致。系统会停止该类规则，不会用估算值替代。

**为什么周末没有扫描？**
周期任务只在交易日 09:30–11:30、13:00–15:00 运行。午休、节假日和停市不请求行情。

**模型不可用会怎样？**
盘中告警完全由代码计算。收盘复盘会降级成数字型报告，仍然保存和展示，但不会虚构行情原因。

**为什么“有色金属”没有价格？**
它是申万一级行业分类，代表一组相关上市公司，并不是一只可以直接交易的股票，所以没有单一股价。要看价格，请在关注页选择具体股票。

**为什么选了某一天却显示“没有数据”？**
常见原因是当天休市、股票尚未上市，或真实数据源暂时没有返回记录。系统不会生成估算价格，可以稍后重试。

**为什么没有买卖建议？**
本工具定位为朋友自用的信息与风险提醒，不是证券投资咨询产品。

**真实通知失败怎么办？**
先到“系统状态”看渠道是否“待配置”，再检查服务端 Secret 名称。日志只记录脱敏状态，不记录密钥。
